# Tag

Tags är etiketter som kan användas för att kategorisera element eller kommunicera framsteg, status eller process. Tags kan ge användaren en snabb överblick över innehållet. Använd tags när innehållet har många kategorier och det är lämpligt att kunna skilja mellan dem. Tags är statiska etiketter; de kan inte klickas på.

<Controls />

## Användning

```tsx
import { Tag } from '@digdir/designsystemet-react';

<Tag>Beta</Tag>;
```

### Rekommendationer

- Använd korta etiketter för enkel skanning. Helst ett ord. Max två ord om det behövs för att beskriva statusen och skilja den från andra taggar.
- Använd stor bokstav i början och små bokstäver för resten.
- Använd ord och begrepp som är igenkännbara för lösningen, undvik obekanta och främmande ord.
- Tags bör vanligtvis placeras efter titeln eller namnet på det tillhörande elementet.
- Tags bör inte placeras tillsammans med knappar, då det kan förvirra användaren. Överväg placeringen och genomför användartest om möjligt.
- Om du vill använda flera tags tillsammans bör du använda en `<ul>` med en `<li>` för varje tagg för att säkerställa att taggarna läses upp separat som en lista och inte som en mening.

### Ej lämpliga för

- Tag är inte lämplig som länk till andra sidor (använd hellre länk).
- Tag är inte lämplig för åtgärder (använd hellre [button](/docs/altinn-button--docs)).
- För filtrering där användaren kan filtrera data och endast se element inom önskad kategori kan du använda Chip-komponenten.

# Card

Med `Card` kan vi framhäva information eller uppgifter som hör ihop. Korten finns i flera varianter och kan innehålla text, bild, textfält, knappar och/eller länkar.

<br />
<Primary />
<Controls />

## Så använder du `Card`

```tsx
import { Card } from '@digdir/designsystemet-react';
<Card>
  <Card.Block>Titel</Card.Block>
  <Card.Block>Innehåll</Card.Block>
  <Card.Block>Valfri fotnot</Card.Block>
</Card>;
```

Använd flera `Card.Block` om du vill dela upp kortet med skiljelinjer eller lägga in bilder eller video som fyller hela ytan.
Observera att innehåll inte kan placeras direkt i `Card` om du använder `Card.Block`; då ska allt innehåll vara inuti en av kortets `Card.Block`-sektioner.

### Klickbart
- `Card` kan användas som navigeringskort för att ta användarna vidare till en annan sida.
- Om kortet ska länka till en annan sida kan du lägga en länk i kortets titel. Hela kortet blir då automatiskt klickbart, men skärmläsare får en [bättre användarupplevelse än om hela kortet var en länk](https://adrianroselli.com/2020/02/block-links-cards-clickable-regions-etc.html).
- Om kortet endast innehåller lite text kan hela kortet bli en länk eller knapp genom att använda `asChild`.

## Riktlinjer för `Card`

`Card` passar för att gruppera innehåll eller funktionalitet som du vill särskilja från resten av innehållet.

Om kortet är en länk med bild ska bilden **inte** vara informativ, och `alt`-attributet ska vara tomt.

Ett kort ska uppfattas som en självständig, avgränsad enhet. Innehållet i kortet ska vara helhetligt och relaterat till ett enda ämne eller en destination.

Kortet ska kunna stå ensamt utan att vara beroende av omgivande element för kontext. Det ska ge tillräckligt med information för att förstå det presenterade innehållet utan externa referenser.

### Undvik

- att använda för många kort på en sida, det kan skapa förvirring
- att blanda för många olika typer av innehåll i ett enda kort

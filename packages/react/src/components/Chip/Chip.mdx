# Chip

`Chip` är små, interaktiva komponenter som låter användarna styra hur de vill se innehåll. Till exempel kan de användas för att filtrera kategorier i ett sökresultat och visa vilka filter som är aktiva.

- Använd inte `Chip` för att märka innehåll med kategori/ämne som inte är interaktivt. Använd istället [`Tag`](/docs/komponenter-tag--docs)-komponenten om kategori/ämne inte är klickbart.

- Använd inte `Chip` som aviseringar för antal eller status. Använd istället [`Badge`](/docs/komponenter-badge--docs)-komponenten.

## Så använder du `Chip`

```tsx
import { Chip } from '@digdir/designsystemet-react';

// som radio
<Chip.Radio name="my-chip" defaultChecked>Du använder Chip-komponenten!</Chip.Radio>;
<Chip.Radio name="my-chip">Du använder Chip-komponenten!</Chip.Radio>;

// som checkbox
<Chip.Checkbox name="my-chip" defaultChecked>Du använder Chip-komponenten!</Chip.Checkbox>;

// som removable
<Chip.Removable>Du använder Chip-komponenten!</Chip.Removable>;

// som button
<Chip.Button>Du använder Chip-komponenten!</Chip.Button>;
```

## Exempel

<br/>
### `Chip.Checkbox` 
`Chip.Checkbox` kan användas som ett alternativ till `Checkbox`.

### `Chip.Removable`
Vi använder `Chip` med kryss när vi vill att användarna ska kunna ta bort valda värden. Det kan till exempel vara filter som är valda, eller värden som användarna själva har skrivit in i ett textfält.

Denna komponent innehåller ett kryss som indikerar att filtret kan tas bort. Det är viktigt att lägga till `aria-label` om innehållet i Chip inte förklarar att det kan tas bort. I detta exempel har `aria-label="Radera godis"` lagts till.

## Riktlinjer för `Chip`

`Chip` låter användarna styra hur de vill visa innehåll. `Chip` kan visas på en eller flera rader och anpassar sig till det utrymme de får.

Undvik att placera `Chip` vertikalt. Välj istället [`Checkbox`](/docs/komponenter-checkbox--docs) eller [`Radio`](/docs/komponenter-radio--docs) om du vill ha alternativ under varandra.

### Passar för att
- filtrera information, till exempel i listor eller tabeller
- visa vilka filter som är valda

### Passar inte för att
- menyer, som sidomenyer eller huvudmenyer
- icke-klickbara kategorier/ämnen (använd istället [`Tag`](/docs/komponenter-tag--docs))
- aviseringar för antal eller status (använd istället [`Badge`](/docs/komponenter-badge--docs))

<br/>

## Text i komponenten
Chips bör ha så få ord som möjligt, helst bara ett eller två.

<br/>

## Tillgänglighet
Om delar av en sida uppdateras samtidigt som användarna gör något på sidan, måste vi se till att användare med skärmläsare får information om detta. Vi kan använda en `ARIA live region` för det, som kan visa antal träffar eller andra ändringar.
